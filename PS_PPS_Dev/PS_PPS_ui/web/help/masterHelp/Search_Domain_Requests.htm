Search Domain Requests

This screen is used to manage the PENDING domain requests that have been entered into the system.  This includes both new item requests and modifications to existing items.  Only those new domains that require a second approval to create will show on the request list and only modifications to attributes that require a second review will show on the request list.  Not all user roles can see all of the checkboxes.  For example, the PPS Second Approver Role can only see the Marked for PSR PEPS Second Reviewer checkbox.

Search for a Pending Request A user action that a PEPS Item (OI, Product Item, or NDC Item) addition or modification be considered by the PNM for updates within the PEPS enterprise database.

1.       Select the type of request to search for.

2.       Click the “Search” button.

 

The search criteria consist of a series of checkboxes.  Those checkboxes are described below.

Request Type

Description

Pending Addition

These are requests that are pending 1st approval.  They are generally requests that have been put into the system by Local Managers.  For the PPS-N Version A variant of some artifact; later versions of an artifact typically expand on earlier versions. Major revisions will mean the version number is incremented by one or more. Minor revisions will be signified by the addition of a decimal number in tenths or hundredths. 1.0 version of the code, there will not be any requests in this state.

Pending Second Approval Addition

These are Addition requests that were submitted by a PPS National manager. They require a different manager to approve them before they are complete.

Pending Second Approval Modification

These are Modification requests that were submitted by a PPS National manager. They require a different manager to approve them before they are complete.

All Requests

This will list all Pending requests.

Under Review

This will list all the requests that have been put into an Under Review state.

Marked for PSR

These are the requests that the National Managers have determined a PPS Second Approver should approve.  They are the ones where a PSR name has been entered.

Change Requests

Change Requests do not exist in the PPS National only version of the code.  They will be introduced when PPS-L is activated.  They allow a local user to request a change to attributes they cannot modify directly.

Not Last Reviewer

This field will filter the search results so you see only those requests that you did not submit or do the last review.

 

 

 

The Search button will execute the search and display the Pending Requests table. The Name column in the Pending Requests field is hyperlinked.  Clicking on the Name field will take the user to the pending item request page.

The result of the search will display the Pending Requests tab which contains a table with 14 columns.  If the search returned no matches to your filter settings, then PPS-N will display “Nothing found to display” below the table.  Try choosing a different filter setting.

When results are displayed in the table, the first column will show the Name of the Product or Orderable Item A generic type representing one conceptual entity consisting of a collection of data fields. For example, a medication may contain the following collection of data fields: Product Name, Generic Name, Trade Name, Dosage Form, NDC, etc. Conceptual entities include Medications, Supplies, and Investigationals., or the number for an NDC National Drug Code.  It is hyperlinked so you can click the link to view the request.
