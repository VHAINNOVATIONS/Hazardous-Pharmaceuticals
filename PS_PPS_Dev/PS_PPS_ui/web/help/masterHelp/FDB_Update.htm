FDB Update

The First Databank Drug Information Framework A set of assumptions, concepts, values, and practices that constitute a way of describing a real situation. is updated on a periodic basis.  A background job runs periodically that will query the FDB First DataBank database to see if any existing FDB products have been updated since the last update was executed.  For more information on the background job see the help files for Manage Application/System Data.

The FDB Update table displays Packaged Drugs that have been updated in the FDB DIF Drug Information Framework since the last update occurred and were changed that could not be automatically made to the PPS-N EPL Enterprise Product List because the changes did not meet specific requirements.  Not all FDB changes are logged in this file, only those that the NDF National Drug File Managers have determined could have potential impacts on VA Department of Veterans Affairs operations are displayed.

FDB Add Results Fields

·         Ndc:  This field is hyperlinked and will display the appropriate NDC National Drug Code.  Note:  This field may be blank if the update affects an FDB Dispensable Drug but not a specific FDB Packaged Drug.

·         VA Product Name A unique name assigned to each drug product in the National Drug files.: The field is hyperlinked and will display the VA product that is being effected by the update.

·         Fdb Generic Name A non-proprietary name assigned to an item (e.g., drug, supply).:

·         Rx Prescription Message A collection of data in a standardized format that can be sent to or received at a given destination.: This is a message indicating original value, the new value and the attribute that was updated.

·         Date Submitted:  This is the date the item was updated in the FDB DIF database.

Select All button: Selects all the checkboxes for the visible rows in the table.

Delete button: Permanently removes the selected items from the FDB Update table.
