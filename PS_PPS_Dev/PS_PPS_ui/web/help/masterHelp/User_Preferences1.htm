User Preferences

The User The requestor, individual, or organization that specifies, or may assist with specification, and typically accepts the project deliverables. There may be multiple users on a given project. Preferences page allows users to specify individual settings for their user experience within the application.

Save User Preferences

Upon completing your necessary changes to the page’s form, click the Set Preferences button.  If you do not want to save your preferences, click the Undo Changes button.

Date / Time Format

This section of the page allows the user to select their date and time display format.  The user can pick from eight date formats and two time formats. 

Date formats:

·         1/9/12 – non-zero-padded month and day with two-digit year

·         1/9/2012 – non-zero-padded month and day with four-digit year

·         1/09/2012 – non-zero-padded month, zero-padded day with four-digit year

·         1/09/12 – non-zero-padded month, zero-padded day with two-digit year

·         01/09/12 – zero-padded month and day with two-digit year

·         01/09/2012 – zero-padded month and day with four-digit year

·         09-Jan-2012 – zero padded day, three-letter short form of month spelled out with four-digit year

·         Mon, Jan 9, 2012 – three-letter short form of day of week and month spelled out, non-zero-padded day with four-digit year.

WithinTime formats:

·         1:00 PM PEPS Manager – standard U.S. time designation with AM or PM (before or after noon)

·         13:00 – 24-hour time designation

Default Login Page

This section of the page allows the user to specify which page they would be directed to after a successful login.  The initial value is the Home page, alternatively, the user may elect to be directed to the [Simple] Search page instead.

Table Display Format

This section allows the user to specify the number of results which may appear on each page of search results.  This value is restricted to values no less than 10 and no more than 100.

FDB Filter Settings

This section relates to settings dealing with FDB First DataBank-related material.

FDB Status Code Format

Available Status Code Formats:  Active, Retired, Replaced, Inactive, Unassociated.

Other Settings

The checkboxes within this portion deal with various other FDB settings that do not fit into a single broad category.  Filters available: Devices, Single Ingredient, Obsolete Drugs, Private Labelers, Repackagers.

FDB RX OTC Over-the-Counter Format

Display both (BOTH), Over the Counter (OTC) or Prescription An Outpatient medication order written on a standard blank issued by a regulatory body. An Outpatient order can be in a written format also on VAF 10-1158 Doctor’s Order sheet. Currently, DEA requires controlled substances to be submitted to Pharmacy in a written format with a “wet signature.” (RX)

FDB Name Type Format

Display both, Brand-only or Generic-only

FDB Packaged Drug Format

Display both, Equivalent Package A set of routines, files, options, templates, security keys, screens, bulletins, functions, help frames, forms, blocks, objects, protocols, dialogues, list templates, windows, and namespaces according to DBA requirements that function as a unit. Drug-only or Package Drug-only.

FDB Phonetic Search Format

Display Literal, Literal and Phonetic or Phonetic

FDB Search Methods Format

Display Begins with Each String, Begins with Literal, Contains Each String, or Contains Literal
