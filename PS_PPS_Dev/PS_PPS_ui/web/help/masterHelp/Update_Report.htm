Update Report

The First Databank Drug Information Framework A set of assumptions, concepts, values, and practices that constitute a way of describing a real situation. is updated on a periodic basis.  A background job runs periodically that will query the FDB First DataBank database to see if any new products have been added to FDB since the last update.  For more information on the background job see the help files for Manage Application/System Data.

The Update Report displays Packaged Drugs that have been updated in the FDB DIF Drug Information Framework since the last update occurred and were then automatically updated in the PPS-N EPL Enterprise Product List because the attributes updated met the criteria to auto update.  The purpose of this report is to show the PPS-N users what PPS-N NDC National Drug Code items were automatically updated.  The managers need not take any additional actions.

Added Report Fields

·         Ndc:  This field is hyperlinked to show the NDC item in the PPS-N EPL.

·         VA Department of Veterans Affairs Product Name A unique name assigned to each drug product in the National Drug files.:  This is the name of the VA Product item in the PPS-N EPL.

·         Fdb Generic Name A non-proprietary name assigned to an item (e.g., drug, supply).:

·         GcnSeqNo:

·         Rx Prescription Message A collection of data in a standardized format that can be sent to or received at a given destination.:  This message contains the attribute that updated and the value to which it has been updated.  (Note: the old value can be seen by clicking on the NDC hyperlink and then looking at the audit history record.).

·         Date Submitted:  This is the date the item was added to the PPS-N EPL.

Select All button: Selects all the checkboxes for the visible rows in the table.

Delete button: Permanently removes the selected items from the Added Report table.
