Migration Control/Migration Report

The Migration Summary is a report showing the status of the last migration.  Each of the files that are migrated are shown in a separate row along with the metrics of the migration.  Each file is hyperlinked so the user can see the details of the migration for a particular file.   A couple items to note form the report.

Ø  The manufacturers and Package A set of routines, files, options, templates, security keys, screens, bulletins, functions, help frames, forms, blocks, objects, protocols, dialogues, list templates, windows, and namespaces according to DBA requirements that function as a unit. Types are not linked since they are not migrated but added as new ones are discovered from the NDC National Drug Code file. 

Ø  The Orderable Items file will show zero records if the user did not choose an Orderable Item A generic type representing one conceptual entity consisting of a collection of data fields. For example, a medication may contain the following collection of data fields: Product Name, Generic Name, Trade Name, Dosage Form, NDC, etc. Conceptual entities include Medications, Supplies, and Investigationals. File in the Configure Migration section.  Orderable Items are then created ‘on the fly’ as needed by new products being added to the system.

Ø  The duplicates Not Migrated numbers are not errors in the system and will not show up in the Error Details.  Duplicates are not migrated by design but they are counted for accounting purposes.
