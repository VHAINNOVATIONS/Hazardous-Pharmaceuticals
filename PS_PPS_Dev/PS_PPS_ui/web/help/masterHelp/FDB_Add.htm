FDB Add

The First Databank Drug Information Framework A set of assumptions, concepts, values, and practices that constitute a way of describing a real situation. is updated on a periodic basis.  A background job runs periodically that will query the FDB First DataBank database to see if any new products have been added to FDB since the last update.  For more information on the background job see the help files for Manage Application/System Data.

The FDB Add table displays Packaged Drugs that have been added to FDB DIF Drug Information Framework since the last update occurred and could not be automatically added to the PPS-N EPL Enterprise Product List because the packaged drug did not meet specific requirements.

FDB Add Results Fields

·         Ndc:  This field is hyperlinked and will display additional FDB Data fields when selected.

·         GcnSeqNo

·         Package A set of routines, files, options, templates, security keys, screens, bulletins, functions, help frames, forms, blocks, objects, protocols, dialogues, list templates, windows, and namespaces according to DBA requirements that function as a unit. Size

·         Package Type

·         Manufacturer Specifies the vendor or company that produces the product.

·         Fdb Generic Name A non-proprietary name assigned to an item (e.g., drug, supply).

·         Label Name

·         Additional Description

·         Creation Date:  This is the date the item was added to the FDB DIF database.

Select All button: Selects all the checkboxes for the visible rows in the table.

Match Results button: Displays the Match Results screen with the selected Ndcs.

Delete button: Permanently removes the selected items from the FDB Add table.
