Migration Start

This screen is broken into three sections. 

Backup Current Database

This section will allow the user to export the current attributes for the VA Department of Veterans Affairs orderable items, products and NDCs to an output CSV Comma Separated Value file for user in a later migration effort.   It is useful in maintaining the changes the user has made to the system during the transition efforts.

When the user presses the Start Download button, the system will begin collating the data and saving the information out to files on the server.  At this point, the data is not being saved to a user machine but to an intermediate location on the server.  It could take several hours for the download process to finish.   While the download process is occurring, the migrator can check on the status of the download by pressing the Status button.  This will show the current status of the download.  Once the download of all three files is complete, the Save Exports button will re-appear. The Save Exports button can be used to save each of the three types of files to a directory on the browser client’s file system.

 

Export Domain Mapping

This section will allow the user to export and import the information from the FDB First DataBank and VA domain mapping tables.   It is useful in maintaining the changes the user has made to the system during the transition efforts.

The Export Domain Refers to data elements used by PEPS items; for example: Manufacturer, Standard Med Route. Usually domain in this context refers to a domain entry, not necessarily the entire domain. Mapping button will allow the user to export the current domain mappings to a csv file.   

The Browse button will allow the user to navigate to a previously exported file. 

The Start Import button will import the domain mapping file chosen with the Browse button into PPS-N EPL Enterprise Product List.

Configure Migration

This section will allow the user to choose the correct import files to use and to start the migration process. The only mandatory file is the NDC National Drug Code file.  The NDC file contains all the NDCs that will be migrated.  The Orderable Item A generic type representing one conceptual entity consisting of a collection of data fields. For example, a medication may contain the following collection of data fields: Product Name, Generic Name, Trade Name, Dosage Form, NDC, etc. Conceptual entities include Medications, Supplies, and Investigationals. and Product file are used in conjunction with date from the National Drug File to ensure proper migration.  Each file is chose independently with the appropriate Browse button.   These files are generally the files that were exported use the Backup Current Database Option.

The Start Migration button is used to start the migration.
